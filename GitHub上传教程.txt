Arco 上传到 GitHub 教程

说明
---

这个项目可以上传到 GitHub，作为本地部署的语音转文字网页工具。

建议只上传代码和教程，不上传模型文件。

不要上传：

checkpoints/
.venv/
音频文件
.DS_Store

这些已经在 .gitignore 里忽略。


第一步：在 GitHub 网页创建空仓库
---

1. 打开 https://github.com/new
2. Repository name 填：

arco

3. 选择 Public 或 Private
4. 不要勾选 Add a README file
5. 不要勾选 Add .gitignore
6. 点击 Create repository

创建完成后，GitHub 会给你一个仓库地址，类似：

https://github.com/你的用户名/arco.git


第二步：在终端进入项目文件夹
---

cd /Users/daphnischloe/Desktop/arco


第三步：初始化 git
---

git init
git branch -M main


第四步：确认不会上传模型
---

git status --short

正常情况下，你不应该看到 checkpoints/ 或 .venv/ 出现在待提交列表里。

如果想再确认一次，可以运行：

git check-ignore -v checkpoints .venv


第五步：提交代码
---

git add .
git commit -m "Initial local speech-to-text web app"


第六步：连接 GitHub 仓库
---

把下面命令里的“你的用户名”换成你的 GitHub 用户名：

git remote add origin https://github.com/你的用户名/arco.git

例如：

git remote add origin https://github.com/daphnischloe/arco.git


第七步：上传
---

git push -u origin main


如果 GitHub 要求登录
---

现在 GitHub 通常不支持直接用账号密码 push。

推荐方式：

1. 按终端提示用浏览器登录；或
2. 使用 GitHub Personal Access Token。

如果终端要求 Password，这里通常要填 token，不是 GitHub 登录密码。


以后修改代码后怎么上传
---

每次改完后运行：

cd /Users/daphnischloe/Desktop/arco
git status --short
git add .
git commit -m "Update app"
git push


别人拿到项目后怎么运行
---

别人 clone 项目后运行：

cd arco
bash scripts/run.sh

然后打开：

http://localhost:8000/ui

模型会下载安装到对方自己的：

checkpoints/huggingface/hub


注意
---

这个项目适合作为“本地部署 speech-to-text 网页端”上传：

- 前端：自定义本地网页
- 后端：FastAPI
- 模型：Whisper large-v3 + Qwen3-ASR 0.6B
- 缓存：项目内 checkpoints，但不上传到 GitHub

如果以后模型越来越多，仍然建议只上传代码，不上传模型。
